## 🔥 Сценарии восстановления

### Сценарий 1: Полный отказ сервера

**Симптомы:**
- Сервер недоступен
- Все сервисы не отвечают
- SSH недоступен

**Действия:**
1. Проверить статус сервера в панели хостинга
2. Попытаться перезагрузить через панель
3. Если не помогает - восстановить из snapshot
4. Развернуть на новом сервере (см. процедуру ниже)

**Время восстановления:** 1-2 часа

---

### Сценарий 2: Потеря базы данных

**Симптомы:**
- PostgreSQL не запускается
- Ошибки подключения к БД
- Данные повреждены

**Действия:**
1. Остановить все сервисы
2. Проверить логи PostgreSQL
3. Попытаться восстановить из WAL
4. Если не помогает - восстановить из backup
5. Запустить сервисы

**Время восстановления:** 30-60 минут

---

### Сценарий 3: Переполнение диска

**Симптомы:**
- Ошибки записи файлов
- Сервисы падают
- Disk usage 100%

**Действия:**
1. Немедленно очистить temp файлы
2. Удалить старые логи
3. Проверить orphan файлы
4. Увеличить размер диска (если нужно)

**Время восстановления:** 15-30 минут

---

### Сценарий 4: Компрометация безопасности

**Симптомы:**
- Подозрительная активность
- Несанкционированный доступ
- Утечка данных

**Действия:**
1. **НЕМЕДЛЕННО** отключить сервер от сети
2. Сменить все пароли и ключи
3. Проверить логи на признаки взлома
4. Восстановить из чистого backup
5. Провести аудит безопасности
6. Уведомить пользователей (если нужно)

**Время восстановления:** 2-4 часа

## 🔧 Процедуры восстановления

### Процедура 1: Восстановление из backup

```bash
# 1. Остановить сервисы
docker-compose down
# или
systemctl stop metacleaner-*

# 2. Найти последний backup
ls -lh /var/backups/metacleaner/

# 3. Восстановить БД
./scripts/restore_database.sh /var/backups/metacleaner/metacleaner_YYYYMMDD_HHMMSS.sql.gz

# 4. Проверить целостность
python scripts/health_check.py

# 5. Запустить сервисы
docker-compose up -d
# или
systemctl start metacleaner-*

# 6. Проверить логи
docker-compose logs -f
# или
journalctl -u metacleaner-bot -f
```

---

### Процедура 2: Развертывание на новом сервере

```bash
# 1. Подготовка сервера
ssh root@new-server
apt update && apt upgrade -y

# 2. Установка зависимостей
git clone https://github.com/your-repo/metacleaner_bot.git
cd metacleaner_bot
bash deploy/setup.sh

# 3. Восстановление конфигурации
# Скопировать .env с безопасного хранилища
scp backup-server:/secure/.env .env

# 4. Восстановление БД
# Скачать последний backup
scp backup-server:/var/backups/metacleaner/latest.sql.gz .
./scripts/restore_database.sh latest.sql.gz

# 5. Запуск сервисов
docker-compose up -d

# 6. Обновление DNS
# Изменить A-запись на новый IP

# 7. Проверка
python scripts/health_check.py
```

---

### Процедура 3: Экстренная очистка диска

```bash
# 1. Проверить использование
df -h
du -sh /app/temp/* | sort -h

# 2. Очистить temp файлы
rm -rf /app/temp/uploads/*
rm -rf /app/temp/processed/*

# 3. Очистить старые логи
find /app/logs -name "*.log" -mtime +7 -delete

# 4. Очистить Docker
docker system prune -af --volumes

# 5. Очистить старые backups
find /var/backups/metacleaner -name "*.sql.gz" -mtime +30 -delete

# 6. Проверить результат
df -h
```

---

### Процедура 4: Откат к предыдущей версии

```bash
# 1. Остановить сервисы
docker-compose down

# 2. Откатить код
git log --oneline -10  # Найти нужный коммит
git checkout <commit-hash>

# 3. Откатить миграции БД (если нужно)
python -m alembic downgrade -1

# 4. Пересобрать контейнеры
docker-compose build

# 5. Запустить
docker-compose up -d

# 6. Проверить
python scripts/health_check.py
```

## 🧪 Тестирование плана

### Ежемесячные тесты

**1. Тест восстановления из backup (1-е число месяца)**
```bash
# На тестовом сервере
./scripts/restore_database.sh /var/backups/metacleaner/latest.sql.gz
python scripts/health_check.py
```

**2. Тест развертывания на новом сервере (15-е число месяца)**
- Развернуть на чистом сервере
- Восстановить из backup
- Проверить все функции

**3. Тест экстренной очистки (каждую неделю)**
```bash
# Симуляция переполнения диска
python scripts/health_check.py
# Если disk usage > 80%, запустить очистку
```

### Документирование тестов

После каждого теста заполнить:
- Дата и время теста
- Кто проводил
- Результаты (успех/неудача)
- Время восстановления
- Проблемы и улучшения
